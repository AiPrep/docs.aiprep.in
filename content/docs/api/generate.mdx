# Generate Video

Create a new video generation job from a text query.

```
POST /api/v1/generate
```

## Headers

- `X-API-Key` (string) — Your API key (use this **or** Authorization)
- `Authorization` (string) — JWT token as `Bearer <token>`

## Request Body

- `query` (string | null) — Text query for video generation (5–5000 chars). Will be summarized if longer than 500 chars.
- `orientation` (string, default: `"landscape"`) — Video orientation: `"landscape"` (16:9) or `"portrait"` (9:16)
- `language` (string, default: `"hindi"`) — Narration language: `"hindi"`, `"english"`, `"hi"`, or `"en"`
- `duration` (integer | null, default: `2`) — Video duration in minutes: `1`, `2`, or `3`
- `mode` (string, default: `"video"`) — `"video"` generates a video, `"text"` generates instant response only
- `chat_id` (string | null, optional) — Associate this job with an existing chat session. If omitted, a new chat is created.
- `images` (string[] | null, optional) — Base64-encoded images (`data:image/png;base64,...`). Max 5 images, 10MB each.
- `cf_turnstile_token` (string | null, optional) — Cloudflare Turnstile token for bot protection. Required for web users, skipped for API key auth.

## Response `200 OK`

```json
{
  "job_id": "abc123-def456-ghi789",
  "chat_id": "chat_xyz789",
  "status": "processing",
  "message": "Video generation started",
  "video_url": null,
  "duration": null,
  "cached": false,
  "instant_response": null,
  "images_metadata": null
}
```

If the query matches a previously generated video, the response returns immediately with `cached: true` and a `video_url`.

## Response Fields

- `job_id` (string, required) — Unique job identifier. Use this to track progress.
- `chat_id` (string | null) — Chat session ID for follow-up messages.
- `status` (string, required) — Job status: `"processing"`, `"completed"`, `"cached"`.
- `message` (string, required) — Human-readable status message.
- `video_url` (string | null) — Video URL (populated immediately if cached).
- `duration` (integer | null) — Video duration in seconds (if cached).
- `cached` (boolean, default: `false`) — `true` if the result was served from cache.
- `instant_response` (object | null) — Instant educational content (if available from cache).
- `images_metadata` (object | null) — Metadata about processed images.
